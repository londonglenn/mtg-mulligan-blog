Learning Mulligan Decisions in Magic: The Gathering

A data-driven model for predicting whether to keep or mulligan an opening hand

Tool Repo Model Repo

Overview

Mulligan decisions in Magic: The Gathering are high-impact choices made at the very start of the game, where players must evaluate the strength of their opening hand with limited information. Despite their importance, these decisions are usually guided by informal heuristics rather than data-driven methods.

In this project, I model mulligan decisions as a probabilistic prediction problem:

P(Keep | Hand)

Using a custom-built data collection tool, I gathered real player decisions and represented each hand using structured features such as land count, card identity, mana curve, and play/draw context.

I trained and evaluated multiple models, including logistic regression, XGBoost, and LightGBM, using 5-fold cross-validation. All models achieved strong performance, with ROC-AUC above 0.90, indicating that mulligan decisions can be predicted reliably.

The most important finding is that simple, interpretable features — especially land count — capture most of the decision signal. More complex models provide only marginal improvements, suggesting that mulligan decisions are driven primarily by mana consistency.

More broadly, this project shows that even complex strategic decisions can often be approximated using compact, interpretable representations, enabling practical decision-support tools for competitive play.

The Decision Problem

Mulligan decisions are one of the first and most important choices in a game of Magic: The Gathering. Given an opening hand of seven cards, a player must decide: keep the hand or mulligan and redraw with fewer resources.

This is a high-stakes tradeoff. Keeping a weak hand can lead to an unplayable game, while mulliganing sacrifices card advantage and long-term resources.

Why This Is Hard

At first glance, this decision seems simple — many players follow rules like “2–3 lands is good.” But in reality, mulligan decisions are much more complex.

The game involves:

  • Hidden information (future draws are unknown)
  • Randomness (card order and outcomes vary)
  • A massive number of possible hand combinations

Because of this, most players rely on heuristics rather than formal reasoning. These heuristics work, but they are difficult to analyze, compare, or generalize across different decks and matchups.

The Ideal Decision (But Infeasible)

In theory, the correct decision would compare:

E[Win | Keep, S] vs E[Win | Mulligan, S]

where S represents the current hand and game context.

However, computing this exactly is impossible in practice due to:

  • The combinatorial number of possible hands
  • Uncertainty about future draws
  • The complexity of full game outcomes

As a result, players rely on intuitive approximations of this comparison.

Our Approach

Instead of trying to compute the optimal decision directly, we take a different approach: learn from real player behavior.

We model the decision as:

P(Keep | S)

This treats observed player decisions as noisy but informative signals of the underlying decision boundary.

The key challenge is representing a hand in a way that a model can understand. Each hand contains complex interactions between:

  • Mana availability (land count)
  • Card composition
  • Mana curve
  • Play vs draw context

What This Project Does

  • Builds a dataset of real mulligan decisions using a custom tool
  • Designs a structured, interpretable feature representation of hands
  • Trains both linear and nonlinear models
  • Uses ablation studies to understand what actually matters

What Has Been Tried Before?

Modeling decision-making in Magic: The Gathering is not new—but most prior work focuses on different parts of the game or uses very different approaches.

Reinforcement Learning (Full Game Simulation)

One major approach is reinforcement learning, where agents learn to play the entire game through simulation, self-play, and search.

While powerful, these methods are:

  • Computationally expensive
  • Highly complex to implement
  • Focused on full-game strategies rather than specific decisions

As a result, they do not directly address isolated decisions like mulligans.

Heuristic and Strategy-Based Methods

Another approach uses handcrafted rules and strategy metrics to evaluate opening hands. For example, some methods measure how well a hand supports a predefined game plan.

These approaches can work well in specific cases, but they:

  • Depend heavily on manual design
  • Do not adapt easily across decks
  • Do not learn from real player data

Representation Learning (Card Embeddings)

Some research focuses on learning representations of cards using neural networks, particularly for drafting and deck-building tasks.

These methods improve generalization across cards and formats, but they target deck construction rather than in-game decisions like mulligans.

Why This Project Is Different

Instead of simulating the game or designing rules, this project learns directly from real player decisions.

We model mulligans as a supervised learning problem:

P(Keep | Hand)

This approach has several advantages:

  • Focuses on a single high-impact decision
  • Uses real player behavior instead of simulated play
  • Produces interpretable results

Key Takeaway

While Magic is an extremely complex game, this work shows that individual decisions like mulligans may have a much simpler structure.

Instead of requiring full-game modeling, we can isolate and understand decision points using compact, data-driven representations.
