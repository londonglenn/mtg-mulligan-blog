Learning Mulligan Decisions in Magic: The Gathering

A data-driven model for predicting whether to keep or mulligan an opening hand

Tool Repo Model Repo

Overview

Mulligan decisions in Magic: The Gathering are high-impact choices made at the very start of the game, where players must evaluate the strength of their opening hand with limited information. Despite their importance, these decisions are usually guided by informal heuristics rather than data-driven methods.

In this project, I model mulligan decisions as a probabilistic prediction problem:

P(Keep | Hand)

Using a custom-built data collection tool, I gathered real player decisions and represented each hand using structured features such as land count, card identity, mana curve, and play/draw context.

I trained and evaluated multiple models, including logistic regression, XGBoost, and LightGBM, using 5-fold cross-validation. All models achieved strong performance, with ROC-AUC above 0.90.

The most important finding is that simple, interpretable features — especially land count — capture most of the decision signal.

The Decision Problem

Mulligan decisions are one of the first and most important choices in a game of Magic: The Gathering. Given an opening hand of seven cards, a player must decide: keep the hand or mulligan.

This is a high-stakes tradeoff. Keeping a weak hand risks losing the game, while mulliganing sacrifices resources.

Why This Is Hard

  • Hidden information
  • Randomness
  • Huge number of possible hands

The Ideal Decision (But Infeasible)

E[Win | Keep, S] vs E[Win | Mulligan, S]

This is impossible to compute exactly, so players rely on intuition.

Our Approach

P(Keep | S)

Instead of solving the game, we learn from real player decisions.

What This Project Does

  • Collects mulligan decisions
  • Builds feature representations
  • Trains ML models
  • Analyzes what matters

What Has Been Tried Before?

Prior work focuses on full-game simulation, heuristics, or card representations— but not directly on mulligan decisions from player data.

Reinforcement Learning

Powerful but complex and focused on full gameplay.

Heuristics

Simple but not generalizable.

Representation Learning

Useful for drafting, not mulligans.

Why This Work Is Different

This project learns directly from real player decisions.

How the Model Works

P(Keep | S)

The model predicts the probability of keeping a hand.

Decision Rule

Keep if P(Keep | S) > τ

Learning Setup

Binary classification:

Keep = 1, Mulligan = 0

Trained using log loss.

Models

  • Logistic Regression
  • XGBoost
  • LightGBM

Data Collection

Custom tool generates hands and records player decisions.

Features

  • Land count
  • Card identity
  • Mana curve
  • Play/draw

Key Insight

Decisions are dominated by mana consistency.

Decision ≈ f(mana) + g(cards)
